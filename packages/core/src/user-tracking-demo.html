User tracking allows you to sync a user's "last viewed" timestamp across devices and sessions via your API. When a user views the changelog, the widget POSTs their timestamp to your server. On subsequent visits, the widget GETs the timestamp from your API to show only new updates.

üîç Testing Instructions

1. Open your browser's developer console to see the logging output:

  • Watch for user tracking API calls (they will fail in demo with mock data)
  • See the fallback to localStorage when API is unavailable
  • Check for the error message when parsing invalid JSON user data

2. Click the badges to open the changelog panel:

  • The badge count should update to show only new updates
  • The widget will attempt to POST to the API (will fail gracefully in demo)
  • localStorage is updated regardless of API success/failure

3. Test localStorage fallback:

  • Click "Clear localStorage" to reset the last viewed timestamp
  • Refresh the page - the badge count should reset
  • Click "Log State" to see the current store state in console
üí° Note: In this demo, API calls are mocked and will not actually connect to a server. The widget gracefully falls back to localStorage-only tracking. In production, connect to your real API endpoints.
